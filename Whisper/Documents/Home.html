Whisper Home Page

Whisper is a general purpose Mac/Win32 C++ application framework. It's the successor to the Mac framework Raven. Like Raven Whisper is a modern framework that takes advantage of templates, multiple inheritance, STL, and exceptions. It's also designed around the notion of Design by Contract and includes numerous debugging tools. There are a number of things I want to achieve with Whisper:

  • I want to make it as easy as possible to write cross platform apps. Ideally this would involve merely a recompile. (At my last company we essentially achieved this using a more specialized framework than Whisper).
  • I don't want to slight either platform: Whisper should be just as attractive to a Windows programmer as to a Mac programmer.
  • I've looked at and used a number of different application frameworks. For the most part they're rather disapointing: most of them don't provide enough functionality, make poor use of abstractions, have weak error handling, and don't take full advantage of C++. You can find more details about these criticisms here.
  • I would like for Whisper to be better than even the platform specific frameworks. Despite the flaws I alluded to above this is a tough job: the other framework vendors have far more resources and they're not hampered by supporting multiple platforms. However I believe that Whisper can achieve this goal in time.
  • Writing any framework is a big job let alone a cross platform framework. My hope is that if I provide a good start others will join in so that Whisper can evolve into a really sweet framework.

CVS

You can browse the current source by pointing your browser at SourceForge and following the "CVS Repository" link. You can also use MacCVS Pro or WinCVS to download the latest source. Note that there may well be problems with the code that's checked into CVS. If you find a problem please let me know and I'll fix it as soon as I can.

The CVS hostname is "cvs.whisper2.sourceforge.net", the root is "/cvsroot/whisper2", and the main module is "Whisper" (there's also an "Extras" module). User name should be "anonymous" with no password.
